Reporting problems or asking questions

If you bump into problems using the Shasta assembler, or if you have questions or suggestions, feel free to submit an issue on GitHub. An effort will be made to respond to issues rapidly. However there is currently only one person doing this, and therefore there can occasionally be periods where issues are not processed for one/two weeks.

Please use one issue per topic when possible:

  • If a new problem occurs, create a new issue rather than appending to an old one. Ok to append to an existing issue for discussion of the same problem for which the issue was originally filed.
  • If two different problems arise on the same run, please file two separate issues.
