Current status

Functionality

In its current implementation, Shasta uses long reads to perform a mostly haploid assembly. Small heterozygous bubbles in the assembly graph are collapsed, and only the largest heterozygous bubbles are kept.

Release

An initial release 0.1.0 of the Shasta assembler is available.

Performance and cost

The Shasta assembler is functional and is capable of performing a human assembly at typical coverage in a few hours on a large machine with an appropriate configuration, using as input long reads from Oxford Nanopore flowcells.

For example, at coverage around 60x, an AWS x1.32xlarge instance, with 128 virtual processors and 1952 GB of memory, completes the assembly in about 5 hours. The compute cost of such an assembly is around $20 at AWS spot market or reserved prices.

The Shasta assembler, although not designed or optimized for smaller genomes, can also assemble bacterial genomes in minutes on a typical laptop (e. g. 1-2 minutes for an E. coli assembly).

Accuracy

An extensive error analysis and comparison with other long read assemblers is available here. Overall, Shasta assembly quality is comparable or better than assembly quality achieved by other, less performant long read assemblers.

Assembly parameters

Default Shasta assembly parameters are optimized for coverage around 60x, and require reconfiguration if using much lower or much higher coverage.

Output

The Shasta assembler creates output in FASTA and GFA 1.0 formats. It also includes an http server and a Python API that can be used to access and visualize the data structures created during assembly. This can be useful for testing or debugging, or for studying assembler behavior at locations of interest.

Platforms

Basic assembly functionality is provided by the Shasta executable which runs on most current 64-bit Linux plus macOS and Windows. The Shasta executable is statically linked, so it has no dependencies and does not require installation. It can be used immediately upon download, subject to setting its execute permission bit.

Extended functionality, including a Python API that enables Python scripting and an http server that can be used to inspect assembly details, are only available on Ubuntu 16.04 and Ubuntu 18.04. A script to install the necessary dependencies is provided.

Functionality to run Shasta under Docker is also provided.

Documentation

Usable documentation for the Shasta executable is provided in the source tree at shasta/docs, and in the install tree at shasta-install/docs. Documentation for the latest code on GitHub is also made available online on GitHub Pages.

The documentation also covers less extensively the functionality provided by the Shasta http server and by the Shasta Python API.

Development

The Shasta assembler is undergoing active development and is expected to improve in accuracy, performance, and functionality. As an open source project, it is open to external contributions.
