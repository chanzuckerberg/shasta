Quick start

It is expected that the first Shasta release will become available in early May 2019. When that happens, it will become very easy for users to get started using Shasta for assembly on a variety of platforms including most current 64 bit Linux platforms, macOS, and Windows. This can be done in three commands - more details below:

wget https://github.com/chanzuckerberg/shasta/releases/download/0.0.0/shasta-linux
chmod ugo+x shasta-linux
./shasta-linux --input input.fasta

See below for more information, including some small changes required for macOS and Windows.

Before the first release becomes available

Before the first release becomes available, users are forced to build the code from source and run the assembly using the more complex procedures described in the old documentation. This requires a Linux machine running Ubuntu 16.04 or Ubuntu 18.04 and root privilege.

When the first release becomes available

When the first release becomes available, users can simply download from GitHub a highly portable executable that has no dependencies and requires no installation or set up, except for setting the execute permission bit.

Linux

  • Download the Linux static executable shasta-linux for your selected release. You can use your browser or the following command, replacing x.y.z with the release you want to use:
    wget https://github.com/chanzuckerberg/shasta/releases/download/x.y.z/shasta-linux
  • Use this command to set its permission bits:
    chmod ugo+x shasta-linux
  • Use this command to run your assembly, making sure to specify the correct path to the location where you stored the executable, or to add that location to your PATH environment variable:
    ./shasta-linux --input input.fasta
    You can specify multiple input FASTA files, if necessary. On a typical laptop, this will run in minutes for a bacterial genome. For a human size assembly, AWS instance type x1.32xlarge is recommended. It is usually available at a cost around $4/hour on the AWS spot market and should complete the human size assembly in a few hours, at coverage around 60x.
  • Assembly output will be created in a new directory named ShastaRun. Output includes the assembly in FASTA and GFA 1.0 formats in files:
    • ShastaRun/Assembly.fasta
    • ShastaRun/Assembly.gfa
This will work on most current 64 bit Linux distributions. It was tested on at least the following Linux distributions for the x86_64 architecture, but it is likely that it will run on any Linux distribution for x86_64 that uses a current kernel.
  • Ubuntu 16.04 and 18.04
  • Linux Mint 18.3
  • CentOS 7.6
  • Debian 9
  • Fedora 29

Note that the procedure above does not require root privilege, unless some non-default options are used when invoking the executable. Those non-default options are, however, necessary to achieve maximum performance.

macOS

Use the Linux directions, but change the name of the executable from shasta-linux to shasta-macOS.

Windows

  • Install Windows Subsystem for Linux (WSL).
  • Working in a Linux shell created using WSL, follow the directions for Linux above.
